# Fashion Digest

## Overview
글로벌 주요 패션 매체에서 흥미로운 기사 10개를 수집하고, 각 기사의 핵심 요약과 업계 시사점을 정리하여 마크다운 파일로 저장한다.

## Execution Workflow

### Phase 1: 병렬 검색 (6개 소스 동시 실행)

아래 6개 검색을 **단일 메시지에서 병렬로** 실행한다:

1. `WebSearch: "site:wwd.com fashion news 2026"` — WWD
2. `WebSearch: "site:businessoffashion.com latest news 2026"` — Business of Fashion
3. `WebSearch: "site:voguebusiness.com news 2026"` — Vogue Business
4. `WebSearch: "site:bloomberg.com fashion luxury industry 2026"` — Bloomberg
5. `WebSearch: "site:ft.com fashion luxury brands 2026"` — Financial Times
6. `WebSearch: "global fashion industry news trends 2026"` — 일반 검색

### Phase 2: 기사 선별 (10개)

수집된 결과에서 다음 기준으로 10개를 선별한다:

- **최신성**: 가장 최근 기사 우선
- **다양성**: 아래 카테고리에서 고르게 선택
  - 브랜드 전략 / M&A / IPO
  - 지속가능성 / ESG
  - 테크 / AI / 디지털
  - 럭셔리 시장 동향
  - 스트리트 / 컬처
  - 유통 / 리테일 변화
- **흥미도**: 업계 임팩트가 크거나, 반전이 있거나, 새로운 트렌드 신호인 기사

중복 URL은 제거한다.

### Phase 3: 각 기사 분석

선별된 각 기사에 대해 `WebFetch`로 본문을 가져온 뒤 분석한다:

| 항목 | 내용 | 분량 |
|------|------|------|
| 핵심 요약 | 무슨 일이 일어났는가 | 3-4줄 |
| 시사점 | 왜 중요한가, 업계에 어떤 영향인가 | 2-3줄 |
| 흥미 포인트 | 이 기사를 선택한 이유 | 1줄 |

### Phase 4: 마크다운 파일 저장

오늘 날짜를 확인하고 `Write` 도구로 저장한다.

**저장 경로**: `fashion-research/YYYY-MM-DD-fashion-digest.md`

**파일 포맷**:

```markdown
# Fashion Digest — YYYY-MM-DD

> 오늘의 글로벌 패션 이슈 10선

---

## 1. [기사 제목](URL)
**출처**: WWD | **날짜**: YYYY-MM-DD | **카테고리**: 브랜드전략

**핵심 요약**
...3-4줄...

**시사점**
...2-3줄...

**흥미 포인트**: ...한 줄...

---

## 2. ...

(3~10 동일 포맷 반복)

---

*Generated by fashion-digest skill*
```

### Phase 5: HTML 웹페이지 생성

Phase 4의 마크다운 저장 직후, 동일 데이터로 HTML 웹페이지를 생성한다.

**저장 경로**: `fashion-research/YYYY-MM-DD-fashion-digest.html`

#### 5-1. 썸네일 이미지 수집

각 기사 URL에 대해 `WebFetch`로 `og:image` 메타태그를 추출한다:
- 성공 시: 해당 이미지 URL 사용
- 실패(페이월·404) 시: 카테고리별 Unsplash fallback URL 사용

- 폰트: Google Fonts — `Playfair Display` (헤딩) + `Inter` (본문)

#### 5-3. 주의사항
- `onerror` 속성으로 이미지 로드 실패 시 자동 fallback 처리
- 카드 순서는 마크다운과 동일하게 유지
- HTML 파일은 마크다운과 **같은 폴더**에 저장: `fashion-research/YYYY-MM-DD-fashion-digest.html`
- 완료 후 사용자에게 두 파일 경로를 함께 안내

### Phase 6: 영어 버전 생성

Phase 4 완료 직후, 각 기사의 **영어 요약**을 생성한다. 별도 검색 없이 Phase 3에서 수집한 내용을 영어로 재작성한다.

각 기사마다:
| Field | Content | Length |
|-------|---------|--------|
| `English Summary` | What happened — concise factual recap | 3-4 sentences |
| `Implications` | Why it matters — industry/market impact | 2-3 sentences |
| `Why This Matters` | One-line highlight — the most compelling angle | 1 sentence |

영어는 자연스러운 비즈니스 저널 문체 (BoF, Bloomberg 수준)로 작성한다. 직역이 아닌 의역.

### Phase 7: 이메일용 바이링구얼 HTML 생성

**저장 경로**: `fashion-research/YYYY-MM-DD-fashion-digest-email.html`

이메일 HTML은 웹용(Phase 5)과 완전히 다른 규칙을 따른다:

#### 이메일 HTML 규칙 (반드시 준수)
- **inline CSS 전용**: `<style>` 태그 금지. 모든 스타일을 `style=""` 속성으로 인라인 처리 (Gmail이 `<style>` 블록 제거)
- **외부 폰트 금지**: Google Fonts 링크 금지. `font-family: Georgia, serif` 등 시스템 폰트만 사용
- **최대 너비 600px**: 이메일 클라이언트 표준
- **테이블 레이아웃**: CSS grid/flexbox 대신 `<table>` 구조 사용 (호환성)
- **이미지 alt 필수**: 이미지 차단 환경 대비

#### Phase 7 주의사항
- 이 파일은 **이메일 발송 전용**. 브라우저에서 열면 스타일이 제한적으로 보이는 것이 정상
- 썸네일은 Phase 5와 동일한 fallback 로직 사용
- 완료 후 생성 파일 3개 경로를 모두 안내: `.md`, `.html`(웹), `-email.html`(이메일)

## Common Mistakes

- **병렬 검색 누락**: 6개 소스를 순차 검색하면 느리다. 반드시 단일 메시지에서 병렬 실행할 것.
- **카테고리 편중**: 럭셔리/브랜드 기사만 고르지 않도록 다양성을 확인할 것.
- **저장 경로 누락**: `fashion-research/` 디렉토리가 없으면 먼저 생성 후 저장.
- **WebFetch 없이 요약**: 검색 결과 스니펫만으로 요약하지 말고 반드시 본문을 가져와서 분석할 것.
- **썸네일 fetch 실패 무시**: og:image 추출 실패 시 반드시 카테고리 fallback URL을 사용할 것. 이미지 없는 카드는 허용하지 않는다.
